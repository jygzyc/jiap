# 漏洞风险评级标准

## 评级等级

| 等级 | 标准 | 典型场景 |
|------|------|----------|
| **CRITICAL** | 远程利用 + 无需用户交互 + 系统级影响 | RCE、system 身份代码执行、全盘数据泄露 |
| **HIGH** | 本地利用 + 权限提升或敏感数据泄露 | Intent 重定向到特权组件、SQL 注入读取敏感数据、clearCallingIdentity 滥用 |
| **MEDIUM** | 本地利用 + 有限影响，或需特定前提条件 | 导出组件越权访问、隐式 Intent 劫持、WebView URL 绕过 |
| **LOW** | 利用条件苛刻或影响有限 | 信息泄露（getType）、点击劫持、本地广播泄露 |

## 可利用性三要素

每个发现必须同时满足以下条件才报告：

1. **可达（Reachable）**：攻击者能触发该代码路径
   - 组件是否导出
   - 是否需要特定权限（攻击者是否具备）
   - 是否需要用户交互

2. **可控（Controllable）**：攻击者能影响关键数据
   - Intent extra 是否可由攻击者控制
   - URI/路径参数是否可注入
   - 是否可通过 AIDL/Messenger 发送任意消息

3. **有影响（Impact）**：造成安全后果
   - 敏感数据泄露（密码、Token、个人信息）
   - 权限提升（普通应用获得 system 权限）
   - 任意代码执行
   - 拒绝服务（仅框架服务报告，应用层不报告）

## 评级决策树

```
是否可达？
├── 否 → 不报告
└── 是 → 是否可控？
    ├── 否 → 不报告
    └── 是 → 是否有影响？
        ├── 否 → 不报告
        └── 是 → 影响等级？
            ├── 远程 + 无交互 + 系统 → CRITICAL
            ├── 权限提升 / 敏感数据泄露 → HIGH
            ├── 有限影响 / 需特定条件 → MEDIUM
            └── 信息泄露 / 条件苛刻 → LOW
```

## 加分因素

- **链式利用**：多个漏洞组合可提升等级（MEDIUM + MEDIUM → HIGH）
- **无安装提示**：通过 Deep Link 触发无需安装恶意应用
- **持久化**：漏洞效果持续存在而非一次性
- **绕过缓解**：绕过 Android 新版本的安全防护（如 checkKeyIntent）

## 减分因素

- **需要物理接触**：降低一级
- **需要特定系统版本**：标注版本范围
- **需要用户明确确认**：降低一级
- **仅影响调试版本**：不报告
